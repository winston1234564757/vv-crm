Проблема #1: Конфлікт специфікації та реалізації

DESIGN.md каже: "ніякого glassmorphism, ніяких градієнтів, тверді поверхні"

Але код (commit 430e53d) впровадив: glassmorphism, backdrop-blur, насичену палітру. Виходить розрив між документом і реальністю.

Поточна палітра

Теплий бежевий фон + фіолетовий акцент. Але в різних частинах кольори плавають — немає єдиної системи токенів, яка використовується всюди.

Page
oklch(96% 0.006 60)
Card
oklch(100% 0 0)
Accent
oklch(50% 0.18 290)

Проблема #2: Єдина типографіка, різне застосування

Readex Pro — гарний вибір, але шкала не дотримується послідовно. Десь 1.75rem display, десь 1.25rem headline — але на практиці розміри плавають між сторінками.

Проблема #3: Фрагментовані компоненти

Кожна форма має свою структуру. Таблиці не використовують спільний компонент. Стилі інлайн або через Tailwind — немає єдиного компонентного шару.

  • Форми: окремі компоненти без спільних патернів
  • Таблиці: кожен розділ пише свою
  • Кнопки: не стандартизовані

Пропоную такий підхід до роботи:

  1. Дизайн-токени — зафіксувати кольори, тіні, радіуси в CSS
  2. Компонентна база — спільні Button, Input, Select, Card, Modal
  3. Навігація — рев'ю сайдбару та мобільного меню
  4. Сторінки — по черзі, починаючи з дашборду
